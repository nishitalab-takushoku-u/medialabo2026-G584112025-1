「Webコンテンツ作成」の結果

本テーマについて

Webページの基本的な記述方法とHTML について学ぶ。 具体的に言うと、HTML 文書を作ることによって、Web ページに掲載するコンテンツと，ページの基本的なレイアウトを設定などを作成する。

プロジェクト課題について

この「番組表の検索結果」と題されたプロジェクト課題のHTMLドキュメントは、2022年3月4日に放送されたNHKの「みんなのうた」に関する2件の検索結果を表示する構成になっております。 1件目には須貝智郎や鬼龍院翔が歌唱する楽曲、2件目には東京事変や財津和夫による楽曲がリストアップされています。 そして、それぞれの開始・終了時刻、タイトル、サブタイトル、番組説明といった詳細情報が、HTMLのリストタグや見出しタグを用いて構造的に整理されています。

プロジェクト課題で作成したページ： project.html

数当てゲーム

本演習では、JavaScriptを使って動的に動く「数当てゲーム」を制作しました。このゲームは、コンピュータがランダムに決めた1から10までの正解を、ユーザーが3回以内に当てるものです。 画面のテキストボックスに数値を入力してボタンを押すと判定が走り、外れた場合は「もっと大きい」「もっと小さい」といったヒントがリアルタイムに画面へ表示されます。 プログラム内部では条件分岐（if文）を使い、正解したときの処理や、3回間違えてゲームオーバーになったときの処理、そして4回目以降の入力に対する無効化の制御などを実装しました。 これにより、HTML要素を書き換えるDOM操作や、ボタン操作に反応するイベント処理の基本を学びました。

数当てゲームのページ： kazuate.html

各授業回の学習事項

第1回 WWW と HTML

第1回の授業では,webコンテンツ作る時必要な記号の書き方を学んだ。例えば、番号なしのリストを作りたい時 ul liを使って、番号付きのリストでは ol liを使うとかいろいろなことを知った。 その上、ハイパーリンクの書き方や画像の埋め込みの書き方も学びました。

プロジェクト課題で作成したページ： 演習1-4:HTMLのさまざまな要素

第2回CSSによるWebページの装飾

第2回の授業では、CSSの作り方と役割について学びました。 CSSを使うことで、文字の色や大きさ、背景などを変更でき、Webページを見やすく整えられることが分かりました。 また、HTMLと分けて管理できるため、とても便利だと感じました。

プロジェクト課題で作成したページ： 演習2-3: ウェブブラウザ説明ページの装飾

第3回JavaScriptの基本

第3回の授業では、JavaScriptの基礎とJavaとの違いについて学びました。 JavaScriptを使うことで、ページに動きや複雑な機能を加えられることが分かりました。Javaと似ている部分もありますが、コンパイル不要でエラーでも止まりにくいという独特の性質があります。 その為、デバッガーを活用して正しく制御することが大切だと感じました。

プロジェクト課題で作成したページ： 演習3-6: index.html

第4回DOMとJavaScript

第4回の講義では、HTML要素をJavaScriptから動的に操作するためのデータ構造であるDOMの概念と、ユーザーのボタンクリックなどの操作に応じてプログラムを実行するイベント処理について学習しました。 具体的には、querySelectorで特定の要素を検索して取得する方法や、textContentを用いたテキストの書き換え、input要素のvalueによる入力値の取得を修得しました。

プロジェクト課題で作成したページ： 演習4-10: index.html

第5回イベント処理

JavaScriptによる動的なDOM操作と効率的なページ装飾を学びました。 document.createElement()での要素生成や、insertAdjacentElement()による挿持位置（beforeendやafterend）の指定、setAttribute()による属性値の変更、remove()による一括削除の手順を習得しました。 また、装飾は直接のスタイル指定より、CSS側で定義したクラスをclassList.add()で適用する設計が簡潔で推奨されると理解しました。

プロジェクト課題で作成したページ： 演習5-7: index.html

1.工夫したところ（工夫点） HTML、CSS、JavaScriptの3つの役割をちゃんと分けました。これはすごく大事です。 HTMLを書くとき、ただ文字を置くだけじゃなくて、ul、liとかh1〜h3のタグを正しく使いました。 NHKの番組表データはとても複雑（時間、タイトル、説明など）だから、人間が見てもパソコンが見ても、データの順番や大事なところがすぐ分かるように工夫しました。 「数当てゲーム」を作ったとき、プレイヤーが困らないようにしました。 数字を入れたら、画面にすぐ「もっと大きい」「もっと小さい」とヒントが出るように、JavaScriptでDOMを動かしました。 このとき、JavaScriptの中で直接色やデザインを変えるのは良くないです。だから、CSSで先にクラスを作っておいて、JSのclassList.add()で動かすようにしました。 これでコードがとても見やすくなって、後で直すのも簡単になりました。 2. 苦労したところ、難しかったところ（苦心点） 数当てゲームで「3回だけチャンスがある」という基本のルールは、簡単に書けました。 でも、「3回間違えてゲームオーバーになったら、もうボタンを押しても何も動かさない」という処理がすごく難しかったです。 ゲームが終わっているのに、またボタンを押すとヒントが変わってしまうバグがありました。フラグ（ゲームの状態を覚える変数）とif文の組み合わせが難しくて、原因を見つけるのにたくさん時間がかかりました。 第5回の授業でinsertAdjacentElement()を使いました。 でも、beforeendとかafterendの意味が頭の中でごちゃごちゃになって、思った通りの場所に表示されなくて困りました。 親要素の中に入るか、隣に並ぶかで、画面の形が全然変わってしまいます。 だから、ブラウザのデバッガーを何回も見て、document.createElement()とsetAttribute()を1つずつチェックしました。 JavaScriptはJavaと違ってコンパイルがないから、動かすまでエラーが分かりにくいです。 ちゃんと考えながらデバッグをすることが本当に大切だと分かりました。 3. アピールしたいところ（強調点） 第1回から第5回まで、授業のスピードがとても早かったです。 最初はただの文字、リンク、画像を置く（HTML）だけでした。 そこからCSSで綺麗にして、最後はJavaScriptで動かすところまで、ちゃんとステップアップできました。 自身の大きな強みになったと思います。 特に第4回と第5回が大事でした。 ユーザーが「ボタンをクリックする」というイベントをきっかけにして、JSのquerySelectorでHTMLの場所を見つけて、textContentやvalueでデータを変える、という一連の流れを完璧に理解しました。 4. 感想 全5回の授業とプロジェクトが終わって、Webページを作るのはとても楽しいし、奥が深いなと思いました。 授業の前は、いつも見ているWebサイトがどうやって動くか、少ししか知りませんでした。 でも、自分でHTMLを書いて、CSSでデザインして、JavaScriptで動きを入れることをやってみて、Webページは細かい設計図がある「動くシステム」なんだと感動した。 今まで勉強したJavaの知識（if文や変数など）が役に立ったのは嬉しかったです。 でも、JavaScriptは「コンパイルしなくて、ブラウザですぐ動く」というルールがあって、その自由さと違いにびっくりしました。新鮮な気持ちです。 エラーが出たときに、デバッガーを使って自分で原因を見つけて、コードを直したら画面がリアルタイムで変わるのが、ものづくりみたいで、終わったときにものすごい達成感がありました。とても充実した演習になりました。
